What Serendipity can help with

1 · Define what HubSpot is supposed to be

StrategyScopeSource-of-truth design
"Before optimizing the portal, we need to define the job of HubSpot. Is it admissions, advancement, alumni — or all three as a relationship and communication layer over LLA's other systems?"
What I'm seeing
  • Admissions, advancement, and alumni fields all coexist
  • Only 1 deal and 50 contacts — system isn't actively used yet
  • RENXT integration signal in test contacts
  • No clear definition of which system owns what
The answer worth proposing
  • HubSpot doesn't replace the SIS, tuition, or RENXT
  • HubSpot is the relationship + communication layer
  • Admissions inquiry → enrolled family → alumni → donor
  • Other systems remain authoritative for records of record

Notable quirks worth flagging

Quirk 01
Advancement fields exist, but very few contacts

The portal has gift officer, capacity rating, constituent ID, alumni status, class year — but only 50 contacts. That mismatch points to either a pilot, a stalled import, or an integration that hasn't completed.

Ask Jacoby: "Is this a test portal, a pilot, or is the full constituent data still coming in?"
Quirk 02
Admissions and advancement are mixed together

Fields suggest both prospective-family tracking and donor/alumni tracking on the same contact object. Not bad — but a prospective parent and a major donor shouldn't move through the same lifecycle or reporting logic.

Quirk 03
Only one deal exists

"Koser – Potential New Student" suggests the model might be student/family opportunity as a deal — which could work well if formalized. Right now, there's no real admissions pipeline yet.

Ask Jacoby: "Should prospective families be tracked as deals, contacts, or both?"
Quirk 04
RENXT integration signal in test data

The contact at test-renxt-integration@serendipitytest.dev is a strong tell — someone has been exploring Raiser's Edge NXT / donor data sync. Worth confirming whether RENXT is the system of record for advancement.

Ask Jacoby: "Is RENXT part of the integration plan? Should HubSpot push to it, pull from it, or stay separate?"
Quirk 05
Constituent fields are promising

Constituent ID, constituent type, alumni status, class year, assigned gift officer, capacity rating — these are exactly the right fields for school advancement. The bones are there. The system just needs real data and workflows on top.
